# Личный кабинет и админ-панель

## Личный кабинет пользователя

Пользователь должен один раз настроить:

- название бизнеса;
- нишу;
- цель: заявки, продажи или доход;
- сайт по умолчанию;
- частоту отчетов;
- уровень уведомлений;
- интеграции;
- бизнес-предпочтения.

После этого бот использует сохраненные данные для:

- отчетов;
- AI-рекомендаций;
- карты прибыли;
- задач "что делать сегодня";
- прогноза;
- content AI;
- smart alerts.

Backend:

- `UserWorkspaceSettings`
- `GET /api/account/{telegram_id}`
- `POST /api/account/settings`

## Админ-панель владельца

Показывает:

- пользователей;
- сайты;
- подписки;
- события tracker;
- отчеты;
- подключенные интеграции;
- MRR;
- trial users;
- churn risk;
- ошибки интеграций;
- pending OAuth;
- операционные риски.

Backend:

- `GET /admin/summary`
- `GET /admin/dashboard-data`

Доступ защищается через `ADMIN_TELEGRAM_IDS` и header `X-Admin-Telegram-Id`. Для production нужно заменить это на полноценную owner-auth модель.
